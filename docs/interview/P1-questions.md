## 一、概念题（1—5）

### 1. Tool Calling 与普通 Python 函数调用有什么本质区别？

- 追问 1：模型是否真正执行了 Python 函数？
- 追问 2：为什么不能从普通文本中猜测工具名和参数？

### 2. `bind_tools()` 在 P1 中承担什么职责？

- 追问 1：它为什么不等于“工具已经执行”？
- 追问 2：绑定之后由谁负责循环和终止？

### 3. `AIMessage.tool_calls` 与 `ToolMessage` 如何构成一次协议往返？

- 追问 1：为什么调用 ID 比工具名更适合关联结果？
- 追问 2：工具失败时还需要 ToolMessage 吗？

### 4. 工具参数 Schema 解决什么问题？

- 追问 1：Schema 能否替代 workspace 路径安全检查？
- 追问 2：多余参数为什么也应该被拒绝？

### 5. P1 的四种运行状态分别表达什么？

- 追问 1：工具失败是否一定导致整个运行失败？
- 追问 2：空最终答案为何属于 `invalid_model_response`？

## 二、源码题（6—10）

- 追问 1：参数校验错误在哪里被捕获？

- 追问 1：结果消息按什么顺序进入 messages？

### 9. `WorkspaceGuard` 如何阻止路径逃逸？

### 10. `ScriptedToolCallingModel` 如何证明循环没有真实网络依赖？

- 追问 1：为什么 P0 的普通 Fake Model 不够？
