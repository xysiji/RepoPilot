# RepoPilot P0 面试题

## 一、概念题（1—5）

### 1. Pydantic `BaseSettings` 在 RepoPilot P0 中解决了什么问题？

- 追问 1：为什么 API Key 仍然需要 `SecretStr` 和安全导出双重保护？
- 追问 2：为什么不在模块 import 时创建全局 Settings？

### 2. 什么是 FastAPI Application Factory，它比复杂的模块级全局 `app` 好在哪里？

- 追问 1：Application Factory 如何帮助接口测试隔离？
- 追问 2：它是否意味着任何全局对象都绝对禁止？

### 3. LangChain `BaseChatModel` 在当前设计中承担什么角色？

- 追问 1：为什么工厂内部仍然可以出现具体的 `ChatOpenAI`？
- 追问 2：Fake Model 为什么也必须遵守同一抽象？

### 4. `/health` 与真实模型可用性探测有什么区别？

- 追问 1：为什么当前 Health 不调用模型 API？
- 追问 2：如果未来需要深度就绪探测，应如何避免破坏当前语义？

### 5. `uv.lock` 与 `pyproject.toml` 各自解决什么问题？

- 追问 1：为什么只在 `pyproject.toml` 写宽泛依赖范围还不够可复现？
- 追问 2：为什么项目约束为 `>=3.12,<3.13`？

## 二、源码题（6—10）

### 6. 请按执行顺序讲解 `create_chat_model()`。

- 追问 1：为什么 `model_override` 检查放在 Provider 校验之前？
- 追问 2：函数返回模型对象后，哪里证明它没有主动访问网络？

### 7. `AppSettings.safe_dump()` 为什么同时排除 API Key 和 Base URL？

- 追问 1：`SecretStr` 默认序列化遮蔽是否足以替代字段排除？
- 追问 2：如何为这个约束写回归测试？

### 8. 从 `create_app()` 到 `/health`，Settings 是怎样传递的？

- 追问 1：为什么通过 `app.state` 保存的是实例而不是配置字典？
- 追问 2：如果测试创建两个 App，它们的配置会互相污染吗？

### 9. `HealthResponse` 使用 `extra="forbid"` 有什么价值？

- 追问 1：FastAPI 的 `response_model` 还提供了什么保护？
- 追问 2：新增字段时应修改哪些位置？

### 10. 配置测试如何证明测试不依赖开发机 `.env`？

- 追问 1：环境变量覆盖测试为何要使用 `monkeypatch`？
- 追问 2：构造参数和环境变量冲突时谁优先，测试如何证明？

## 三、设计取舍题（11—15）

### 11. 为什么 P0 最终没有依赖 `langchain` 元包？

- 追问 1：改成 `langchain-core` 加 Provider 集成牺牲了什么？
- 追问 2：如何确认没有把 LangGraph 留在锁文件里？

### 12. 为什么 P0 只支持一个真实 Provider，却仍然需要模型工厂？

- 追问 1：什么时候才值得引入 Provider Registry？
- 追问 2：当前方案如何避免业务代码绑定 OpenAI？

### 13. 为什么不把模型对象在应用启动时立即创建出来？

- 追问 1：延迟构造会把哪些错误推迟到哪里？
- 追问 2：Health 中的 `model_configured` 为什么不是“模型一定可用”？

### 14. 为什么 P0 不实现 daemon、EventBus 或数据库？

- 追问 1：移除 daemon 牺牲了哪些真实能力？
- 追问 2：什么需求出现时才应重新评估后台运行能力？

### 15. 为什么配置边界由 Python 校验，而不能只写在 Prompt 或文档里？

- 追问 1：temperature 超界可能产生什么问题？
- 追问 2：哪些 P0 决策仍属于策略而不是安全边界？

## 四、异常与排错题（16—20）
